# 需求文档：绩效考核报表系统

## 简介

本系统为二手车拍卖平台提供精细化的绩效考核报表功能。系统以车辆为维度，实时追踪从入库到售后的全生命周期数据，自动计算各项成本和绩效毛利，为评估师、门店总监、区域经理、财务人员等角色提供清晰的绩效评估数据。

核心公式：
- 交易总毛利 = 唯普毛利 + 唯普云毛利 + 汇和毛利 + 机拍毛利
- 绩效毛利 = 总交易毛利 - 交付成本(提车费+现场交付成本/空中交付成本+现场过户成本) - 检测成本 - 销售成本 - 数据成本 - 政企场地费 - 收购车成本 - 业务合作费
- 总成本 = 提车费 + 检测成本 + 评估费 + 销售成本 + 收购车成本 + 交付成本 + 过户成本 + 垫款利息 + 业务合作服务费 + 其他成本 + 数据成本 + 政企场地费

## 术语表

- **绩效考核系统（System）**: 二手车拍卖平台的绩效考核报表系统，负责数据查询、计算、展示和导出
- **BMS**: 业务管理系统，作为车辆基础信息和订单数据的上游数据源
- **唯普毛利**: 车辆收益归属唯普，取值字段 weipusy + wpsy_231213，同经管报表同字段名取值
- **唯普云毛利**: 车辆收益归属唯普云，取值字段 wpyunsy，同经管报表同字段名取值
- **汇和毛利**: 车辆收益归属汇和，同经管字段同字段名。现有取值：huihesy、fshhgzsy（佛山汇和广州分公司收益）、hhgzsy（汇和贵州分公司收益）+ 汇和收益：通过钱包流水摘要【衍生服务-其他-结算】提取车牌匹配90天内成交订单（抛仲裁退车）。新增：门店自垫款收益纳入门店的那部分数据 + 零售溢价 + 门店自垫款车辆调账款项
- **机拍毛利**: 车辆收益归属机拍，取值字段 pmzx02；同经管报表同字段名取值
- **交易总毛利**: 唯普毛利 + 唯普云毛利 + 汇和毛利 + 机拍毛利，一台车仅归属一方
- **绩效毛利**: 总交易毛利 - 交付成本 - 检测成本 - 销售成本 - 数据成本 - 政企场地费 - 收购车成本 - 业务合作费，是最终考核指标
- **总成本**: 提车费 + 检测成本 + 评估费 + 销售成本 + 收购车成本 + 交付成本 + 过户成本 + 垫款利息 + 业务合作服务费 + 其他成本 + 数据成本 + 政企场地费
- **门店自垫款车辆**: 门店垫款收购款项的车辆，毛利会清分到门店账户中；毛利调整也需要纳入（调账）
- **零售衍生**: 车辆走零售，门店自拍后再销售，有个衍生单，衍生单的收益 = 成交价 - 合同价
- **冲正行**: 售后发生时系统自动生成的调整记录，冲正项目为交易总毛利、唯普毛利、唯普云毛利、汇和毛利、机拍毛利
- **沉没成本**: 不因售后问题进行冲正的成本项（任何售后场景均不冲回），包含：检测成本、评估费、销售成本、收购车成本、提车费、垫款利息、现场交付成本、空中交付成本、现场过户成本、业务合作服务费、数据成本
- **可冲回项**: 发生售后问题就进行冲正的毛利相关项：交易总毛利、唯普毛利、唯普云毛利、汇和毛利、机拍毛利。注意：售后仲裁退车时，所有成本项（含业务合作服务费）均不做冲正；
- **时机点（Trigger_Point）**: 车辆生命周期中的6个关键数据更新节点：T1入库（唯普库/服务中心库）、T2检测过审、T3成交付款清分后、T4交付型出库、T5过户资料回收完毕、T6售后执行完成
- **检测成本**: 省内85元/台，省外120元/台
- **数据成本（出险查询）**: 26元/次（原28元，已调整）、出险查询、电池查询等多个费用项

- **业务合作服务费**: 待定
- **提车人**: 提车助手中将车辆送入唯普库的操作人，即实际执行提车入库的人员
- **提车类型**: 分为自提（自己送车）和第三方（他人送车，姓名+手机号，在唯普系统不存在）
- **空中交付部门**: 区分两种情况：服务中心（对应具体服务中心名称）和中控（由中控部门统一调度）
- **空中交付跟进人**: 点击资料回收完毕对应的系统账号操作人姓名/台，省外120元/台
- **政企场地费**: 人工核算，需有列展示和导出字段
- **业务合作服务费**: 与订单清算挂钩，清算后立即计算成本。按集团/门店分档计费：理想汽车按合同价分5档（300~3000元/台）、深圳大茂鹏鸿300元/台、利泰集团按月成交台数和起拍价分档（100~400元/台）、鸿蒙智行200元/台。仲裁退车时系统自动生成对冲负数成本（防止重复计费）。售后仲裁退车不做冲正纯占位字段，需有列展示和导出字段
- **业务合作服务费**: 待定
- **评估师**: 负责车辆评估的人员，仅可查看自己名下车辆数据
- **门店总监**: 门店管理者，可查看本门店所有车辆数据
- **区域经理**: 区域管理者，可查看本区域下属门店所有车辆数据
- **汇和财务**: 可以查看汇和垫款车辆 + 12家门店自垫款车辆（待定）
- **唯普财务**: 可查看非汇和垫款车辆数据（待定）
- **副部长**: 事业部管理者，可查看事业部维度数据
- **绩效管理员**: 系统管理员，可查看和管理全部数据
- **数据隔离**: 根据用户角色自动限制可见数据范围的安全机制
- **筛选器（Filter）**: 用户界面中用于缩小数据查询范围的交互控件
- **导出模块（Exporter）**: 负责将报表数据输出为Excel和CSV文件的功能模块

## 需求

### 需求 1：交易总毛利与汇和毛利计算

**用户故事：** 作为财务人员，我希望系统自动计算每辆车的交易总毛利和汇和毛利，以便获得绩效计算的基础数据。

#### 验收标准

1. WHEN T3成交支付完成时，THE System SHALL 根据车辆垫款方（唯普/唯普云/汇和/机拍）直接获取对应毛利值，并汇总计算交易总毛利（交易总毛利 = 唯普毛利 + 唯普云毛利 + 汇和毛利 + 机拍毛利），一台车仅归属一方
2. WHEN 车辆属于12家奥迪门店时，THE System SHALL 按公式 汇和毛利 = 汇和垫款车辆毛利（huihesy + fshhgzsy + hhgzsy）+ 汇和收益（钱包流水摘要【衍生服务-其他-结算】提取车牌匹配90天内成交订单，抛仲裁退车）+ 门店自垫款收益 + 零售溢价 + 门店自垫款车辆调账款项 计算汇和毛利分项
3. WHEN 计算零售衍生收益时，THE System SHALL 按公式 零售衍生收益 = 成交价 - 合同价 计算衍生单收益

### 需求 3：绩效毛利计算

**用户故事：** 作为评估师，我希望系统自动计算绩效毛利，以便清楚了解自己的考核数据。

#### 验收标准

#### 验收标准

1. WHEN 用户点击表格行的查看详情按钮，THE System SHALL 弹出详情框，展示该车辆的基础信息（车牌、VIN、车源类型、合同价、起拍价、成交价、成交日期、买家付款状态、评估师、门店）
2. WHEN 详情框打开时，THE System SHALL 展示财务信息区域，包含交易总毛利、唯普毛利/唯普云毛利/汇和毛利/机拍毛利、综合服务费、各成本项明细（检测成本、评估费、销售成本、提车费、现场交付成本/空中交付成本、现场过户成本、数据成本、政企场地费、业务合作服务费）、总成本、绩效毛利
3. WHEN 详情框打开时，THE System SHALL 按时间顺序展示T1至T6的时机点日志：T1记录入库时间/地点/提车费；T2记录过审时间/检测成本/数据成本/检测员；T3记录成交日期/成交价/起拍价/合同价/评估费/综合服务费/销售成本/收购车成本；T4记录交付型出库类型/时间/交付方式/交付部门/现场交付成本；T5记录过户资料回收时间/过户部门/现场过户成本/空中交付成本/交付部门；T6记录售后时间/售后状态/责任方/额外损失/是否豁免/冲正行
4. WHEN 车辆存在售后记录时，THE System SHALL 在详情框中额外展示售后信息区域，包含售后原因、责任方、额外损失、冲正行详情（交易总毛利、唯普毛利、唯普云毛利、汇和毛利、机拍毛利的冲正金额）
5. THE System SHALL 在详情框底部提供关闭和打印两个操作按钮

### 需求 8：多维度筛选功能

**用户故事：** 作为区域经理，我希望通过多种条件筛选绩效数据，以便快速定位和分析特定范围的车辆数据。

1. THE System SHALL 提供以下筛选条件：订单清分日范围、日期范围、检测过审日期范围、车牌/VIN/VID关键字、合同类型（全部/不限/收购/委拍）、车辆状态（全部/已检测/已成交/已付款/已交付/已售后）、门店名称（支持搜索联想带出）、大区域（支持多选）、小区域（支持多选）。筛选区域支持展开/折叠
2. WHEN 用户选择大区域筛选值后，THE System SHALL 联动更新小区域下拉列表，仅显示所选大区域下的小区域选项。大区域选项来源于基础数据表（含1-广东-广州、1-广东-佛山、2-广东-深圳、2-广东-东莞等共30+个大区）
3. WHEN 用户输入门店名称关键字时，THE System SHALL 提供模糊匹配的自动联想建议列表，支持搜索联想带出门店、车牌/VIN/VID关键字、合同类型（全部/不限/收购/委拍）、车辆状态（全部/已检测/已上拍/已成交/已付款/已交付/已售后）、门店名称（支持多选）、大区域（支持多选）、小区域（支持多选）
2. WHEN 用户选择大区域筛选值后，THE System SHALL 联动更新小区域下拉列表，仅显示所选大区域下的小区域选项
3. WHEN 用户输入门店名称关键字时，THE System SHALL 提供模糊匹配的自动联想建议列表
4. WHEN 用户点击搜索按钮，THE System SHALL 按所有已设置的筛选条件取交集查询并刷新表格数据
5. WHEN 用户点击重置按钮，THE System SHALL 将所有筛选条件恢复为默认值（日期默认为本周）
6. THE System SHALL 仅展示100条数据，超出部分仅可通过导出查看
7. IF 筛选结果为空，THEN THE System SHALL 显示暂无符合条件的数据提示信息

### 需求 9：数据导出功能

**用户故事：** 作为财务人员，我希望将绩效数据导出为文件，以便进行离线分析和财务对账。

#### 验收标准

1. THE System SHALL 支持将当前筛选结果导出为Excel（.xlsx）和CSV（.csv）两种格式
2. WHEN 用户点击导出按钮，THE 导出模块 SHALL 生成包含以下列的文件：序号、车牌、VIN码、VID、确认日期、成交日期、合同类型、评估师、门店全称、检测员、成交价、交易总毛利、唯普毛利、唯普云毛利、汇和毛利、机拍毛利、提车费、检测成本、评估费、销售成本、政企场地费、交付成本、交付部门、现场过户成本、现场过户部门、车辆状态、售后状态、责任方、绩效毛利、绩效工资
3. THE 导出模块 SHALL 按命名规则 报表名_日期范围_导出时间_用户名 生成导出文件名
4. THE 导出模块 SHALL 在导出时自动应用当前用户的权限过滤，确保导出数据不超出用户可见范围

### 需求 10：角色权限与数据隔离

**用户故事：** 作为绩效管理员，我希望系统根据用户角色自动限制数据可见范围，以便保障数据安全和隐私。

#### 验收标准

1. WHEN 评估师登录系统，THE System SHALL 仅显示该评估师名下的车辆数据，并将门店和评估师筛选器锁定为灰显不可编辑状态
2. WHEN 门店总监登录系统，THE System SHALL 仅显示该门店所有车辆数据，门店筛选器锁定为本门店，评估师筛选器可选本门店所有评估师
3. WHEN 区域经理登录系统，THE System SHALL 仅显示该区域下属门店的所有车辆数据
4. WHEN 汇和财务登录系统，THE System SHALL 仅显示汇和垫款车辆和12家门店自垫款车辆的数据（待定）
5. WHEN 唯普财务登录系统，THE System SHALL 仅显示非汇和垫款车辆数据（待定）
6. WHEN 绩效管理员登录系统，THE System SHALL 显示全部车辆数据，所有筛选器均可自由操作
7. THE System SHALL 在数据库查询层面实施行级数据隔离，确保用户无法通过接口绕过权限限制访问超出范围的数据
8. THE System SHALL 对评估师和门店总监角色禁用编辑和删除功能，对区域经理和财务角色禁用删除功能，仅副部长和绩效管理员拥有删除权限

### 需求 11：左侧功能菜单与导航

**用户故事：** 作为系统用户，我希望通过清晰的菜单导航快速访问各功能模块，以便高效完成日常工作。

#### 验收标准

1. THE System SHALL 在左侧提供固定宽度200px的导航菜单，包含以下菜单项：首页、报表中心（经管报表、车源绩效表）、数据查询（高级搜索、数据导出）
2. THE System SHALL 不展示系统管理菜单（用户管理、权限设置、费用标准维护、操作日志）
3. THE System SHALL 将帮助内容（使用指南、字段含义）放到绩效考核表内作为Tab标签页展示
4. WHEN 用户点击菜单项时，THE System SHALL 高亮当前选中的菜单项（绿色背景），并在主内容区加载对应页面
5. THE System SHALL 在顶部提供导航栏，包含系统名称、首页、报表、我的、登出入口
6. WHILE 用户处于任意页面，THE System SHALL 保持左侧菜单和顶部导航栏始终可见

### 需求 12：响应式布局与界面交互

**用户故事：** 作为系统用户，我希望界面在不同屏幕尺寸下均可正常使用，以便在不同设备上查看绩效数据。

#### 验收标准

1. WHILE 屏幕宽度大于1200px，THE System SHALL 以4列网格布局展示筛选区域
2. WHILE 屏幕宽度在1024px至1200px之间，THE System SHALL 以3列网格布局展示筛选区域
3. WHILE 屏幕宽度在768px至1024px之间，THE System SHALL 以2列网格布局展示筛选区域，左侧菜单宽度缩减为160px
4. WHILE 屏幕宽度小于768px，THE System SHALL 以单列布局展示筛选区域，左侧菜单改为顶部水平布局
5. THE System SHALL 对表格区域提供水平滚动支持，确保所有列在窄屏幕下可通过横向滚动查看
6. WHEN 用户将鼠标悬停在表格行上，THE System SHALL 以浅灰色背景高亮该行
7. WHEN 筛选输入框获得焦点时，THE System SHALL 以绿色边框和阴影效果标识当前活跃的输入框
